IdeaSwatch
IdeaSwatch happened while I was waiting for the iphone 6 to come out. I wanted to learn about RESTful API's, so I decided I would try it out. At this point, I had already done a few tutorials on setting up Node servers and working with Mongo, so I decided to make something that might be a little more useful in the context of design. This project was primarily aimed at just learning how API's work, and how Angular routing works. Users can enter information such as a title, blurb, links, and hexes. The idea for this was it could just serve as a quick reference link for people who wanted to share palettes, something that happens a lot in design groups. Ideally, I would have wanted to add some sort of OAuth login, but it wasn't really necessary or in the scope of the project. Got to try out unique URL's and got familiar with RESTful API's!
Technologies Angular.js, HTML, CSS, Mongo, Node.js, Express.js
Lessons First experience dealing with single page application
Decided on a MVP I could get done before the iphone 6 came out
