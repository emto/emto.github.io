Where to start? Well, first matter at hand was deciding how high fidelity I wanted this website to be. Should I include the chat components? Navigation? How much interaction should I include? Given a time constraint of 1 week, where most of the week I'd be unavailable to work freely on this project, I decided to go with a MVP. I had 2 goals that I wanted to accomplish by the end of this project. 1) Learn React 2) Portray the gist of the site So... did someone say React? A year ago I would've ran away at the mention of React. There was no documentation then. :( But here we are, a year later, and there are some scraps of documentation. So let's see how this goes! But wait... do you really need React? Errr.. actually not really. After playing with React, it seems the perks are dynamically loading content. This means node generations can be done incredibly fast, so adding and deleting content is a breeze. For the project, there wasn't any data generation so it really wasn't all that necessary -- however, it did really improve the development process since React allows you to modularize different components of your website. The framework actually worked perfectly with the video player, since there's a browsing section, a nav section, and a chat bar -- all of which need to have their own logic. Singular movie node that takes in props title, duration, genre, and an image link. ...Now for the popovers. Popovers were never my friend.... Oh my god, that was terrible. Implementing popovers wasn’t the hard part, it was figuring out if they should be on the left or right side, based on where the movie is placed on a grid relative to a screen size. Since React renders these individually, the offset values within the node don’t tell me much, so I had to code up some hard math values to figure out where the movie was relative to the screen. If there’s some better way to do this, pls help me. I basically used findDOMNode, and traversed up the parents until I got the document size, then did some voodoo math to find out where our original node is relative to this. Ok, it works. Why does my stupid popover keep flickering? Time to write some weird logic in my render function that actually executes this left or right logic.
